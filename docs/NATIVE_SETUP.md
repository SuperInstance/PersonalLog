# Native Integration Setup Guide

**PersonalLog v1.1 - WASM Performance Extensions**

## Overview

PersonalLog now includes WebAssembly (WASM) extensions for high-performance vector operations. This guide will help you set up the development environment and build the native modules.

## Prerequisites

### Required Software

1. **Rust** (1.70 or later)
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```

2. **wasm-pack** (for compiling Rust to WASM)
   ```bash
   cargo install wasm-pack
   ```

3. **Node.js** (18 or later)
   - Already installed if you can run `npm`

### Verify Installation

```bash
rustc --version    # Should be 1.70+
wasm-pack --version # Should be 0.10+
node --version      # Should be 18+
```

## Quick Start

### 1. Initial Setup

```bash
# Install Rust toolchain (if not already installed)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Install wasm-pack
cargo install wasm-pack

# Navigate to project
cd /mnt/c/users/casey/PersonalLog
```

### 2. Build WASM Module

```bash
# Build the WASM module
npm run build:wasm

# Expected output:
# [INFO]: Checking for WASM features...
# [INFO]: Compiling to WASM...
# [INFO]: Optimizing WASM...
# [INFO]: Done in 45.2s
#    Compiled WASM to native/rust/pkg/personallog_native_bg.wasm
```

### 3. Start Development Server

```bash
# WASM builds automatically, then starts Next.js
npm run dev

# Or manually build then start
npm run build:wasm && npm run dev
```

### 4. Verify WASM is Loaded

Open browser console, you should see:

```
[VectorStore] Native ops loaded: success
[WASM] Loaded successfully (v0.1.0, SIMD: true)
```

If WASM fails to load, you'll see:

```
[VectorStore] Native ops loaded: fallback
[WASM] Failed to load module...
[WASM] This is expected in development if WASM is not built yet
```

## Development Workflow

### Normal Development

```bash
# 1. Make changes to Rust code (native/rust/src/)
# 2. Rebuild WASM
npm run build:wasm

# 3. Start development server
npm run dev

# 4. Test in browser
# Open http://localhost:3002
```

### Testing WASM Changes

```bash
# 1. Edit native/rust/src/lib.rs or vector.rs
# 2. Rebuild
npm run build:wasm

# 3. Restart dev server (Ctrl+C, then npm run dev)

# 4. Check browser console for success/error
```

### Iterating Quickly

During development, you can skip WASM rebuilds by commenting out the build:

```json
// package.json (temporary!)
"scripts": {
  "dev": "next dev -p 3002",  // Skip WASM build for speed
  "build:wasm": "cd native/rust && wasm-pack build ..."
}
```

When done, restore the original scripts.

## Troubleshooting

### Problem: `wasm-pack: command not found`

**Solution:**
```bash
cargo install wasm-pack
```

### Problem: Build fails with "error: linker not found"

**Solution:** Install C build tools:

**macOS:**
```bash
xcode-select --install
```

**Ubuntu/Debian:**
```bash
sudo apt install build-essential
```

**Windows:**
Install [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022)

### Problem: `npm run build:wasm` is very slow

**Solution:** This is normal. First build takes 1-2 minutes, subsequent builds are faster (30-45s).

To speed up:
```bash
# Use Rust's nightly compiler (experimental)
rustup install nightly
rustup default nightly

# Then rebuild
npm run build:wasm
```

### Problem: Browser shows "WASM not supported"

**Solution:** Update your browser:
- Chrome 91+ (for SIMD)
- Firefox 89+ (for SIMD)
- Safari 15.2+ (for SIMD)
- Edge 91+ (for SIMD)

Older browsers will automatically use JavaScript fallback.

### Problem: Changes to WASM code don't appear

**Solution:** You must rebuild WASM after code changes:

```bash
# 1. Rebuild
npm run build:wasm

# 2. Hard refresh browser (Ctrl+Shift+R)
# 3. Or clear browser cache
```

## Integration with Vector Store

The vector store automatically uses WASM when available:

```typescript
// src/lib/knowledge/vector-store.ts

class VectorStore {
  private vectorOps: Awaited<ReturnType<typeof getVectorOps>> | null = null

  async init() {
    // Load WASM ops asynchronously
    getVectorOps().then(ops => {
      this.vectorOps = ops
      console.log('[VectorStore] Native ops loaded: success')
    })
  }

  private cosineSimilarity(a: number[], b: number[]): number {
    // Try WASM first
    if (this.vectorOps) {
      return this.vectorOps.cosine_similarity(a, b)
    }
    // Fallback to JS
    return cosineSimilarityJS(a, b)
  }
}
```

## Adding New Native Operations

### 1. Add to Rust

```rust
// native/rust/src/vector.rs

#[wasm_bindgen]
pub fn my_new_function(input: &[f32]) -> Vec<f32> {
    // Your native code here
    input.iter().map(|x| x * 2.0).collect()
}
```

### 2. Rebuild

```bash
npm run build:wasm
```

### 3. Add to Bridge

```typescript
// src/lib/native/bridge.ts

my_new_function(input: number[]): number[] {
  const input32 = new Float32Array(input)
  try {
    const result = wasm.my_new_function(input32)
    return Array.from(result)
  } catch {
    // JS fallback
    return input.map(x => x * 2)
  }
}
```

### 4. Use in App

```typescript
const ops = await getVectorOps()
const result = ops.my_new_function([1, 2, 3])
```
